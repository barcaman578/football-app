# /new-feature

Build a complete feature for Ballknowledge end to end.

## Instructions

Ask the user:
1. What is the feature called?
2. Describe what it does in 2-3 sentences
3. What pages or components does it touch?
4. What database tables does it need?

Then plan and build the full feature:
1. Database — create or update Supabase tables needed
2. Backend — API routes or server actions needed
3. Components — all UI components needed
4. Pages — create or update pages
5. Wire it all together

Follow all rules in CLAUDE.md throughout. Build in this order:
database → helpers → components → page.

After building, summarise what was created and what the user needs to do manually (e.g. run SQL in Supabase dashboard, add env variables).
